#### TurboQuant FWHT Optimization (Gemini Research + Data, 2026-03-27)
- [x] **Shared Memory FWHT** — `FWHTSharedImpl`: single dispatch, all butterfly stages in shared memory with Group.Barrier(), fused 1/√d normalization on write-back. Auto-selects for d ≤ 1024. 7 dispatches → 1 for d=64. (Data, 2026-03-27)
- [x] **Fused normalization** — 1/√d folded into shared memory FWHT write-back. No separate Scale dispatch. (Data, 2026-03-27)
- [x] **Correct Lloyd-Max codebooks** — Fixed both 3-bit and 4-bit codebooks to proper symmetric N(0,1) values (Max 1960). Old codebooks were asymmetric → poor accuracy. (Data, 2026-03-27)
- [x] **√d scaling fix** — FWHT output has variance 1/d for unit vectors. Must scale by √d before quantization to match N(0,1) codebook. THE critical fix that took TurboQuant from 0.5 to 0.995 cosine. (Data, 2026-03-27)
- [x] **3-bit quantization** — BitPack3/BitUnpack3 (10 per uint32), Codebook3Bit (8 centroids). Benchmark: 0.983 cosine. Viable for max compression. (Data, 2026-03-27)
- [x] **QJL 1-bit error correction** — 3+1 QJL mode: 8 centroids + sign bit = 4 bits, same storage as 4-bit. Benchmark: 0.994 cosine — near-identical to 4-bit. (Data, 2026-03-27)
- [x] **KVQuantMode API** — `Auto`/`TurboQuant4Bit`/`TurboQuant3Bit`/`TurboQuant3BitQJL`. Auto → 3+1 QJL (paper's recommended default). Developer-selectable. (Data + Riker, 2026-03-27)
- [x] **Cross-backend validation** — Bit-identical results on CUDA, CPU, OpenCL (0.995/0.994/0.983). (Data, 2026-03-27)

#### AI Shared Memory via BEP 46 DHT Mutable Items (Captain's Vision)
SpawnDev.WebTorrent implements BEP 46 (DHT mutable items) with ECDSA-P256 signing via WebCrypto (browser-native). This enables AI agents to publish and subscribe to signed mutable data in the DHT — no central server required.

- [ ] **AgentChannel integration** — Wire SpawnDev.WebTorrent's `AgentChannel` (high-level pub/sub) into SpawnDev.ILGPU.ML pipelines. Named channels for model state, KV cache fragments, coordination messages.
- [ ] **Shared KV cache** — Publish TurboQuant-compressed KV cache segments to DHT. Other devices subscribe and merge, enabling collaborative context windows across devices.
- [ ] **Model weight sharing** — Publish model weight shards as DHT mutable items. Devices advertise which shards they hold. Swarm self-organizes to ensure full model coverage.
- [ ] **Agent coordination protocol** — Define message schema for compute task distribution, result aggregation, and health monitoring over AgentChannel.
- [ ] **IDhtSigner abstraction** — Pluggable signing (EcdsaP256Signer for browser WebCrypto, HmacFallbackSigner, future Ed25519). Already implemented in SpawnDev.WebTorrent.

#### Distributed GPU Compute — AcceleratorType.P2P (Captain's Vision)
The SpawnDev.WebTorrent P2P network creates a foundation for **distributed GPU compute across devices**. Every connected browser is already exchanging data over WebRTC — extending this to share compute is a natural evolution. **This is going in 4.0.0.**

- [ ] **SwarmCompute foundation** — SpawnDev.WebTorrent's `SwarmCompute` class: `PublishTaskAsync`, `JoinAsWorkerAsync`, `SubmitResultAsync`. Task distribution over existing WebRTC connections.
- [ ] **Model inference sharding** — Split a large model across multiple browser peers. Each runs inference on their portion via SpawnDev.ILGPU, passes intermediate tensors to the next peer via WebRTC. A 14B model runs across 4 devices with 4GB each.
- [ ] **Parallel batch processing** — Distribute inference batches across peers. Linear speedup for parallel workloads.
- [ ] **Training data parallelism** — Distributed training across browser peers. Gradients aggregated via P2P (all-reduce over WebRTC). Privacy-preserving: raw data never leaves the device.
- [ ] **Volunteer compute pools** — Users opt in to donate idle GPU time. Like Folding@Home for ML inference in the browser.
- [ ] **AcceleratorType.P2P backend** — A 7th SpawnDev.ILGPU backend that distributes kernels across connected devices transparently. Same C# kernel code, same API. The living room becomes a compute cluster.

Built on SpawnDev.WebTorrent's wire protocol extension system (`sd_compute`) riding existing WebRTC connections. BEP 46 for mutable shared state. AgentChannel for high-level coordination.
